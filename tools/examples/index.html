Standalone QR Codes

    

      Each QR code is a self-contained payload — no other codes needed.
      Scan any one with the web reader or the Android app.
    

    

  


  

    
Passphrase-Encrypted Content (SPEC §9 + KDF)

    

      A content code whose hidden override map is unlocked by entering a passphrase
      — no separate key-code QR. The code embeds a random KDF salt (kdf_salt, CBOR key 38)
      and algorithm hint (kdf_alg=1, CBOR key 37); the reader derives the AES-256-GCM key
      via PBKDF2-HMAC-SHA256 (100 000 iterations) when prompted.
      The passphrase for this example is: sunset.
    

    

    

  


  

    
Hidden Content — Key-Code QR (SPEC §9)

    

      A content code whose visible "cover" is a generic placeholder, plus a hidden
      override map — AES-256-GCM encrypted and carried as trailing bytes —
      and a separate key code. Scan either one first: the content code
      decodes immediately to its cover, while the override stays locked until the key code
      is also scanned (in either order), per SPEC §9's "discovery, not declaration": no field
      declares which content a key unlocks, so the reader tries it by trial decryption and,
      on a match, replaces the cover with the hidden fields. The content code's
      cache_id is random rather than content-addressed, so it can't be used as a
      content-equality oracle.
    

    

    

  


  

    
Multi-Code Cache (Manifest + Chunks)

    

      Content too large for one QR (> ~800 bytes) is split into a
      Manifest code plus several Chunk codes.
      Chunks can be scanned in any order — even from different physical
      locations — and TagDrop reassembles and checksums them once complete.
    

    

    

  


  

    
Paper Set (Multi-file)

    

      A paper groups multiple files under one manifest.
      Scan the manifest QR first, then the file QRs in any order.
    

    

    

  


  

    
Mini E-Book (Markdown Pages)

    

      A paper of text/markdown pages — a cover/table-of-contents and three
      chapters — that link to each other with ordinary relative links and share one
      style.css (SPEC.md §7's stylesheet convention). Scan the manifest, then
      every file, then open index.md and tap through: about as close to an
      e-book as a sheet of paper gets.
    

    

    

  


  

    
Multi-Location Trail (Multiple Papers)

    

      Three independent paper manifests — one per "stop" — each with its own icon and
      related hints to its neighbours. Useful for testing what happens when multiple pages
      are scanned at different locations: separate map pins, shared collection grouping, and trail
      navigation hints.
