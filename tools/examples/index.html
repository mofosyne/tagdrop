Multi-Code Cache (Manifest + Chunks)

Content too large for one QR (> ~800 bytes) is split into a Manifest code plus several Chunk codes. Chunks can be scanned in any order — even from different physical locations — and TagDrop reassembles and checksums them once complete.

Sunset Trail story — text/plain, 1426 bytes total, 3 chunks
Cache ID: 81d3adcdf38c1d15
SHA-256 (assembled): 81d3adcdf38c1d15dbb4e9cd3cd554a895d9ae14bf0fccd9da5e67e85d5551fa
① Scan the manifest — any time, before or after the chunks.
② Scan each chunk in any order. TagDrop reassembles and verifies the SHA-256 once all 3 are collected.

Paper Set (Multi-file)

A paper groups multiple files under one manifest. Scan the manifest QR first, then the file QRs in any order.

Root hash: 0b5546d2978c78fc
Navigation link base: tagdrop://LJ1+.867JCDF/test-paper

② Scan each file QR in any order to cache its content.
